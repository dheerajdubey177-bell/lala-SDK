# Lala Runtime ABI Specification (v1.0)

This document specifies the Application Binary Interface (ABI) contract between the code generated by the Lala compiler, the underlying operating system, and the Lala Runtime Library. It serves as the frozen specification for compiler and runtime authors.

## 1. Binary Interface

*   **Architecture**: x86-64 (AMD64)
*   **Object Format**: ELF64 (Linux), PE/COFF (Windows), Mach-O 64 (macOS)
*   **Endianness**: Little-Endian
*   **Word Size**: 64-bit

## 2. Program Entry

The underlying platform invokes a platform-specific entry point (e.g., `_start` on Linux, `main` CRT wrapper on Windows). The Platform Startup is strictly responsible for:
1.  Initializing any runtime data structures (like allocators or stack bounds).
2.  Invoking the Lala entry point: `lala.main()`.
3.  Receiving the integer return value from `lala.main()`.
4.  Executing the system exit call with the received return value.

## 3. Return Value

*   `lala.main()` must return a 64-bit integer.
*   This integer is placed in the `RAX` register before the `RET` instruction.
*   The runtime entry code uses `RAX` as the OS exit code.

## 4. Stack Alignment

*   On entering any function (including `lala.main()`), the stack pointer (`RSP`) must be aligned to a 16-byte boundary.
*   Before issuing a `CALL` to external runtime functions, the caller is responsible for ensuring the 16-byte alignment of the stack immediately before the `CALL` instruction is executed.

## 5. Register Preservation

Lala generated code follows the standard C Calling Convention of the target platform:
*   **System V AMD64 ABI (Linux / macOS)**:
    *   **Arguments**: `RDI`, `RSI`, `RDX`, `RCX`, `R8`, `R9`
    *   **Return**: `RAX`
    *   **Callee-Saved**: `RBX`, `RBP`, `R12`, `R13`, `R14`, `R15`
    *   **Caller-Saved**: `RAX`, `RCX`, `RDX`, `RSI`, `RDI`, `R8`, `R9`, `R10`, `R11`
*   **Windows x64 Calling Convention**:
    *   **Arguments**: `RCX`, `RDX`, `R8`, `R9`
    *   **Return**: `RAX`
    *   **Callee-Saved**: `RBX`, `RBP`, `RDI`, `RSI`, `R12`, `R13`, `R14`, `R15`
    *   **Caller-Saved**: `RAX`, `RCX`, `RDX`, `R8`, `R9`, `R10`, `R11`
    *   **Shadow Space**: The caller must reserve 32 bytes (4 words) of shadow space on the stack immediately above the return address.

*(For v1.0, the compiler will assume System V AMD64 ABI universally for internal functions, but must adapt to the platform ABI when calling standard C libraries if we use libc as a backing for the runtime).*

## 6. Exception / Panic Contract

*   The compiler inserts checks (e.g., bounds checks, division by zero) directly into the generated code.
*   When a check fails, the compiler executes a `CALL lala_panic`.
*   `lala_panic` is an external runtime symbol. It does not return.
*   The runtime is responsible for printing a generic failure message and terminating the process with a non-zero exit code.
