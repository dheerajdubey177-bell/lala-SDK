# Low-Level IR (LIR)

## Purpose
The LIR breaks down complex language statements into basic instruction sets executed within Basic Blocks. It models the program as a Control Flow Graph (CFG).

## Structure
- **ProgramIR**: Contains all `FunctionIR` instances.
- **FunctionIR**: Contains `BasicBlock` instances and tracks the entry block.
- **BasicBlock**: Contains `LIRInstruction` operations and maintains connections to sibling blocks.
- **LIRInstruction**: Granular operations such as `LOAD_CONST`, `LOAD_VAR`, `STORE_VAR`, `BINARY_OP`, `JUMP`, and `BRANCH`.

## Inputs
Generated by the `IRBuilderPass` iterating over the AST.

## Outputs
The CFG stored in `CompilerContext.lir`.

## Ownership
`ir/lir.py` defines the schema; `ir/builder.py` constructs it.

## Invariants
- Control flow uses Basic Blocks linked via `BRANCH` and `JUMP`.
- LIR is platform-agnostic (it has no knowledge of C++).
