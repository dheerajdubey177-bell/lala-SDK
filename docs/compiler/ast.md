# Abstract Syntax Tree (AST)

## Purpose
The AST is the structural representation of grammatically valid Lala source code.

## Inputs
Tokens generated by the `Lexer`.

## Outputs
A tree of `ASTNode` objects (e.g., `ProgramNode`, `IfNode`, `FunctionDeclNode`).

## Ownership
The AST is generated by `Parser` and stored in `CompilerContext.ast`.

## Invariants
- The AST is **immutable** after the `Parser` completes.
- Semantic passes (Symbol Resolution, Type Checking) must traverse the AST and store metadata externally, rather than modifying the nodes.
- Syntax errors must be caught before or during AST construction. The AST represents *only* syntactically valid constructs.
