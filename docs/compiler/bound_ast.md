# Bound AST

The Bound AST is generated by the `BoundTreeBuilder` after the Semantic Pipeline has completed its analysis.

## Invariants
1. **Fully Resolved**: All identifiers are resolved to `Symbol`s.
2. **Fully Typed**: All expressions have an explicit canonical `Type`.
3. **Runtime Bound**: External calls are bound to `RuntimeID`s, and built-ins to `IntrinsicID`s.
4. **Independent**: The Bound AST does not require `CompilationContext` side-tables; all semantic knowledge is embedded in the `BoundNode`s.

## Node Hierarchy
* `BoundNode`
  * `BoundStatement`
    * `BoundDeclaration` (e.g., `BoundVariableDecl`, `BoundFunctionDecl`)
    * `BoundIfStatement`
    * `BoundWhileStatement`
    * `BoundReturnStatement`
  * `BoundExpression`
    * `BoundLiteralExpression`
    * `BoundVariableExpression`
    * `BoundIntrinsicCall`
    * `BoundRuntimeCall`
